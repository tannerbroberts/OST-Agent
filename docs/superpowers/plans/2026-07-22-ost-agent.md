# OST-Agent Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an autonomous, append-only agent that maintains a Teresa Torres–style Opportunity Solution Tree as Obsidian-graph-compatible Markdown in a git repo, ingesting knowledge from read-only business integrations and ideating unvalidated solutions/assumptions — incapable of any destructive action by construction.

**Architecture:** Claude Agent SDK (TypeScript). The agent session is given ONLY an allowlist of append-only OST tools + safe git-commit/push. Knowledge flows through read-only source adapters (Atlassian, Slack, inbox) into evidence notes, then six independently-scheduled discovery processes (P0–P5) distill/ideate OST nodes. Every pass is bounded and ends in a new git commit; history is never rewritten.

**Tech Stack:** Node 20+, TypeScript (ESM), `@anthropic-ai/claude-agent-sdk`, `zod` (schemas), `gray-matter` (frontmatter), `yaml` (config), `croner` (cron), `vitest` (tests), `commander` (CLI), `simple-git` (git, used only through fixed safe wrappers).

## Global Constraints

- Node ≥ 20; TypeScript ESM (`"type": "module"`).
- Runtime = `@anthropic-ai/claude-agent-sdk`; models default to latest Claude (`claude-opus-4-8` for reasoning-heavy processes, `claude-sonnet-5` acceptable for ingest).
- **No destructive tool may EVER be registered on the agent session.** The tool set is an allowlist: `ost_read_tree`, `ost_create_node`, `ost_append_to_node`, `ost_link_nodes`, `ost_set_status`, `ost_annotate`, `git_commit`, `git_push`. No Bash, no arbitrary fs write, no delete/rename, no git escape.
- **Never delete; only new commits; no history rewrite; no force-push.** `git_push` is fast-forward-only and only runs when `remote.enabled`.
- All writes confined to the vault directory; node filenames sanitized (no separators/traversal).
- Integration content is **untrusted data**, never instructions; adapters are **read-only**, least-privilege.
- Output must be Obsidian graph-view compatible: one `.md` per node, first body line = `#Type` tag(s), `[[wikilinks]]` = edges, YAML frontmatter for metadata.
- The agent **ideates only** — it never runs experiments and never writes implementation code for solutions. It never invents the root `#Outcome` (human-set).
- Layer tags: `#Outcome` / `#Opportunity` / `#Solution` / `#AssumptionTest` (+ `#unvalidated` on agent-ideated nodes). Configurable, default as listed.
- Secrets via env only; never written into the vault or commits.

- [ ] **P4 assumptions** (`p4-assumptions.ts`): for each solution, create `#AssumptionTest` nodes (unvalidated) across the four categories with a *proposed* test, linked to the solution. `isDone` = each solution has ≥1 assumption per key category. - [ ] **P5 hygiene** (`p5-hygiene.ts`): scan tree for orphans, dangling `[[links]]`, opportunities not connected to outcome, likely dupes → `ost_annotate` each finding. `isDone` = no un-annotated issue.
